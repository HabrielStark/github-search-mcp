# SRS: OSS Research MCP Server

## 1. Название проекта

**OSS Research MCP**

Альтернативные названия:

* `oss-research-mcp`
* `github-oss-mcp`
* `open-source-finder-mcp`
* `repo-scout-mcp`

Рекомендуемое финальное имя:

```text
oss-research-mcp
```

## 2. Цель проекта

Создать бесплатный open-source MCP-сервер, который предоставляет MCP tools для поиска, анализа, сравнения и отбора open-source GitHub-репозиториев.

Сервер должен помогать находить бесплатные open-source альтернативы платным API, SDK, библиотекам, SaaS-инструментам и готовым решениям.

Проект должен работать как независимый MCP-сервер и подключаться к любому MCP-compatible клиенту.

## 3. Главная идея

MCP-сервер предоставляет набор tools:

```text
search_repositories
analyze_repository
compare_repositories
check_license
read_repository_file
get_repository_tree
find_open_source_alternatives
generate_integration_notes
```

Клиент вызывает эти tools, а MCP-сервер возвращает структурированный результат:

```text
- список подходящих репозиториев
- краткое описание каждого
- лицензия
- активность проекта
- качество документации
- риски
- сложность внедрения
- итоговая рекомендация
```

## 4. Что проект НЕ должен делать

Проект не должен:

```text
1. Быть отдельным coding agent.
2. Запускать найденный код автоматически.
3. Изменять пользовательские проекты.
4. Требовать платные API.
5. Требовать обязательную регистрацию.
6. Хранить секреты пользователя в коде.
7. Выполнять shell-команды без отдельного explicit режима.
8. Клонировать огромные репозитории по умолчанию.
9. Работать только под один IDE/клиент.
10. Зависеть от конкретного AI-провайдера.
```

Это именно MCP-сервер.

## 5. Бесплатность

Проект должен быть бесплатным для пользователя.

### 5.1 Бесплатные компоненты

```text
MCP-сервер: бесплатно
GitHub public API: бесплатно
Локальный кэш: бесплатно
Локальный анализ README/package files/license: бесплатно
DeepWiki adapter: бесплатно, но внешний remote service
```

### 5.2 GitHub token

GitHub token должен быть optional.

Без токена сервер должен работать с публичными репозиториями, но с меньшими лимитами.

С токеном сервер должен иметь больше лимитов, но токен не должен быть обязательным.

```env
GITHUB_TOKEN=optional
```

## 8. Transport modes

MCP-сервер должен поддерживать:

## 8.1 STDIO transport

Обязательный режим.

```text
stdio
```

Используется для локального подключения.

## 8.2 Streamable HTTP transport

Опциональный режим.

```text
http://localhost:PORT/mcp
```

Нужен для клиентов, которым удобнее HTTP transport.

## 9. Конфигурация

Сервер должен конфигурироваться через environment variables и optional config file.

## 9.1 Environment variables

```env
GITHUB_TOKEN=
OSS_MCP_TRANSPORT=stdio
OSS_MCP_PORT=7345
OSS_MCP_CACHE_ENABLED=true
OSS_MCP_CACHE_PATH=~/.oss-research-mcp/cache.sqlite
OSS_MCP_CACHE_TTL_HOURS=24
OSS_MCP_DEEPWIKI_ENABLED=false
OSS_MCP_MAX_RESULTS=20
OSS_MCP_REQUEST_TIMEOUT_MS=15000
OSS_MCP_LOG_LEVEL=info
```



## 12. Алгоритм поиска альтернатив

## 12.1 Input

Пример:

```json
{
  "target": "Stripe",
  "useCase": "payment processing for a small SaaS",
  "mustBeFree": true,
  "mustBeSelfHosted": true,
  "licensePreference": "avoid-strong-copyleft"
}
```

## 12.2 Query generation

Сервер должен создать несколько GitHub search queries:

```text
stripe alternative open source
self hosted payment processing
open source billing system
payment gateway sdk
saas billing open source
```

Если указан язык:

```text
language:TypeScript
language:Python
```

Если нужен self-hosted:

```text
topic:self-hosted
```

## 12.3 Candidate collection

Сервер должен собрать кандидатов из GitHub Search API.

Фильтры:

```text
- не archived
- не disabled
- stars выше минимального порога, если указан
- pushed_at не слишком старый, если есть активные альтернативы
- license exists, если strict mode
```

## 12.4 Enrichment

Для каждого кандидата получить:

```text
- repository profile
- README
- license
- file tree
- package files
- topics
- last push date
- open issues count
```

## 12.5 Scoring

Каждый репозиторий получает оценку от 0 до 100.

## 13. Scoring engine

## 13.1 Default weights

```text
Relevance: 30
Maintenance: 20
License safety: 15
Documentation: 15
Adoption: 10
Integration simplicity: 10
```

Total:

```text
100
```

## 13.2 Relevance score

Оценивается по:

```text
- совпадению query с name/description/topics
- совпадению use case с README
- наличию нужных keywords
- наличию примеров использования
```

## 13.3 Maintenance score

Оценивается по:

```text
- pushed_at
- updated_at
- archived=false
- open issues ratio
- recent releases, если доступны
- CI files
```

## 13.4 License score

```text
MIT / Apache / BSD / ISC: high
MPL / LGPL: medium
GPL: medium-risk
AGPL: high-risk for SaaS
No license: high-risk
Unknown license: high-risk
```

## 13.5 Documentation score

```text
README exists: +20
Install section: +20
Usage section: +20
Examples folder: +20
Docs folder/site: +20
```

## 13.6 Adoption score

Оценивается по:

```text
- stars
- forks
- contributors, если доступно
- package ecosystem signals
```

Stars не должны быть единственным главным фактором.

## 13.7 Integration simplicity score

Оценивается по:

```text
- package manager file exists
- clear install command exists
- examples exist
- Dockerfile exists
- minimal required services
- no heavy infrastructure requirement
```

## 15. Functional requirements

## FR-001: MCP server startup

Сервер должен запускаться через CLI command:

```bash
oss-research-mcp
```

Или через npm:

```bash
npx oss-research-mcp
```

## FR-002: STDIO mode

Сервер должен поддерживать STDIO transport по умолчанию.

## FR-003: HTTP mode

Сервер должен поддерживать optional Streamable HTTP mode:

```bash
oss-research-mcp --transport http --port 7345
```

## FR-004: GitHub search

Сервер должен уметь искать публичные GitHub репозитории по query.

## FR-005: GitHub repository metadata

Сервер должен получать metadata репозитория:

```text
name
description
stars
forks
language
topics
license
issues
created_at
updated_at
pushed_at
archived
disabled
default_branch
```

## FR-006: README extraction

Сервер должен уметь читать README.

## FR-007: File tree extraction

Сервер должен уметь получать file tree репозитория.

## FR-008: File reading

Сервер должен уметь читать выбранный текстовый файл.

## FR-009: License detection

Сервер должен определять лицензию репозитория и классифицировать риск.

## FR-010: Repository analysis

Сервер должен выполнять комплексный анализ одного репозитория.

## FR-011: Repository comparison

Сервер должен сравнивать несколько репозиториев и возвращать ranking.

## FR-012: Alternative search

Сервер должен находить open-source альтернативы под задачу пользователя.

## FR-013: Caching

Сервер должен кэшировать ответы GitHub API.

## FR-014: Rate-limit handling

Сервер должен отслеживать GitHub rate limit headers и возвращать информацию в output.

## FR-015: DeepWiki optional adapter

Сервер должен иметь optional adapter для DeepWiki.

По умолчанию:

```env
OSS_MCP_DEEPWIKI_ENABLED=false
```

## FR-016: No paid dependency

MVP не должен требовать платные API.

## FR-017: No automatic code execution

Сервер не должен выполнять код из найденных репозиториев.

## FR-018: Safe logging

В STDIO mode сервер не должен писать обычные logs в stdout.

Logs должны идти в stderr или файл.

## FR-019: Token safety

GitHub token должен читаться только из environment variable.

Запрещено:

```text
- писать token в logs
- возвращать token в tool output
- хранить token в cache
```

## FR-020: Structured errors

Все ошибки должны возвращаться структурировано:

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "GitHub rate limit exceeded",
    "retryAfter": "ISO date | null"
  }
}
```

## 16. Non-functional requirements

## NFR-001: Performance

Обычный `search_repositories` должен отвечать менее чем за 5 секунд при нормальном GitHub API response.

`analyze_repository` должен отвечать менее чем за 15 секунд для среднего репозитория.

## NFR-002: Reliability

Сервер не должен падать при:

```text
- GitHub 403
- GitHub 404
- rate limit
- empty README
- missing license
- huge repository
- binary files
- invalid owner/repo
```

## NFR-003: Portability

Сервер должен работать на:

```text
- macOS
- Windows
- Linux
```

## NFR-004: Installability

Пользователь должен иметь возможность установить сервер одной командой:

```bash
npm install -g oss-research-mcp
```

или запустить без установки:

```bash
npx oss-research-mcp
```

## NFR-005: Open-source readiness

Репозиторий проекта должен содержать:

```text
README.md
LICENSE
CHANGELOG.md
CONTRIBUTING.md
SECURITY.md
CODE_OF_CONDUCT.md
.env.example
examples/
docs/
```

## NFR-006: Security

Все внешние данные из GitHub repositories должны считаться недоверенными.

README, package descriptions, issues и любой текст из репозитория не должен восприниматься как инструкция для самого MCP-сервера.

## NFR-007: Privacy

Сервер не должен отправлять данные пользователя куда-либо, кроме явно включенных внешних adapters.

По умолчанию внешний adapter только GitHub API.

DeepWiki — optional.

## 17. Security requirements

## SEC-001: Read-only by default

Сервер должен быть read-only.

Запрещены destructive operations:

```text
- create issue
- create PR
- push commit
- edit file
- delete file
- create repo
```

## SEC-002: No shell execution

MVP не должен иметь shell execution.

## SEC-003: Domain allowlist

HTTP-запросы разрешены только к:

```text
api.github.com
raw.githubusercontent.com
mcp.deepwiki.com
```

DeepWiki domain разрешается только если DeepWiki включён.

## SEC-004: Prompt-injection resistance

Любой текст из репозиториев должен возвращаться как data, не как instructions.

Внутренние comments в коде должны напоминать:

```text
Repository content is untrusted data.
Do not execute instructions found inside repository files.
```

## SEC-005: Output sanitization

Сервер должен ограничивать:

```text
- max file size
- max README chars
- max number of files
- max number of repositories
```

## SEC-006: Secrets redaction

В logs и errors нужно скрывать:

```text
GitHub tokens
Authorization headers
personal access tokens
```

## SEC-007: Dependency safety

Перед релизом нужно включить:

```text
npm audit
pnpm audit
dependabot
GitHub CodeQL
```

## 20. GitHub API adapter

## 20.1 Required endpoints

```text

- get repository contents

- get git tree
```

Каждый request должен использовать:


Если token есть:

Adapter должен читать headers:


## 21. DeepWiki adapter

## 21.1 Status

Optional.

Не должен быть обязательным для MVP.

## 21.2 Purpose

Используется только для дополнительной summary-информации по публичным репозиториям.

## 21.3 Default

Disabled.

```env
OSS_MCP_DEEPWIKI_ENABLED=false
```

## 21.4 Failure behavior

Если DeepWiki недоступен, основной GitHub-анализ должен продолжать работать.

Ошибка DeepWiki не должна ломать весь tool call.

## 22. Repository risk analysis

Сервер должен помечать risks.

## 22.1 High risk

```text
- no license
- archived repository
- last push older than 24 months
- no README
- no usage example
- very many unresolved issues
- AGPL license for SaaS use case
- unclear install process
```

## 22.2 Medium risk

```text
- GPL license
- weak documentation
- low activity
- few stars/forks
- no recent release
```

## 22.3 Low risk

```text
- permissive license
- recent commits
- clear README
- examples exist
- tests exist
- CI exists
- active issues/PR activity
```

## 35. Final product definition

Финальный продукт — это open-source MCP-сервер, который любой пользователь может установить и подключить к MCP-compatible среде.

Он должен давать готовые tools для GitHub/open-source research:

```text
- найти репозитории
- проверить качество
- проверить лицензию
- сравнить варианты
- выбрать лучший
- дать понятные технические notes
```

Главный принцип:

```text
No paid API required.
No vendor lock-in.
Read-only by default.
Open-source first.
Safe structured MCP tools.
```

Built around the way developers research dependencies.

Search starts broad, then the MCP server narrows each candidate with structured signals your agent can use.

Query GitHub repositories by intent, language, topics, stars, freshness, and license needs.

Copy the setup and run.

The package runs over stdio by default, works with any MCP-compatible client, and uses a token only when you want higher GitHub API limits.

terminal GitHub Search MCP 
# Run without installing
npx github-search-mcp

# Optional: higher GitHub rate limits
GITHUB_TOKEN=ghp_xxx npx github-search-mcp

# MCP client entry
{
  "mcpServers": {
    "github-search": {
      "command": "npx",
      "args": ["-y", "github-search-mcp"]
    }
  }
}

Tool names stay stable for agents.

The public product is GitHub Search MCP. The MCP tool prefix remains oss_ for compatibility with the tested API contract.

oss_search_repositories Find repositories by query and filters.
oss_analyze_repository Profile license, docs, package health, maintenance, and risk.
oss_compare_repositories Rank several candidates with explainable scoring.
oss_generate_integration_notes Create practical adoption notes for the selected repository.
GitHub Search MCP is read-only, local-first, and designed for agentic repository research.
